# WSL2 Waydroid 内核编译项目

## 项目背景与思考

### 方案一：GLM5.1 单 Agent 全流程

我尝试过使用 GLM5.1，让单个 Agent 从头到尾完成整个项目。但结果令人失望：

- **计划、实现、总结三个部分完全对不上**
- 无论怎么要求它维护文档、同步版本，到后面就是"弱智"状态
- LLM 生成的内容它自己都不清楚是怎么回事

这让我思考：有没有一种方案可以解决这个问题？

### 方案二：Kimi K2.5 多任务拆分（当前项目采用）

本项目采用的方法是：

- **使用 Kimi K2.5**（主要是因为不需要排队）
- **提示词、第一份计划、第一次生成、每一次审核修正、运行验证、修复问题都通过新建任务来完成**
- **没有类似方案一的文档维护环节**，每一次任务都要智能体去读取代码文件和提示词文件中的需求内容
- 这种方法对于代码量较小的项目也许还能顶得住，算是一种**粗糙的 rewind**

## 关于 Trae 的吐槽

Trae 过于封闭：

- **智能体的管理**就是一坨屎
- **插件的使用**也是一坨屎

如果不解决这个问题，除非 Trae 能搞到 Claude 新出的那个模型，不然**时间成本远比不上模型成本**。
