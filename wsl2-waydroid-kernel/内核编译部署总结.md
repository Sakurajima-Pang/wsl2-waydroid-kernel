# WSL2 Waydroid 内核编译与部署总结

## 项目概述

本文档总结了在 Windows Subsystem for Linux 2 (WSL2) 上编译和部署支持 Waydroid 的自定义 Linux 内核的完整过程。

## 遇到的问题与解决方案

### 1. 脚本执行问题

#### 问题：脚本意外退出
**现象**：运行 `03-build-kernel.sh` 时在用户确认后直接退出

**原因**：
- `((current_step++))` 在值为 0 时返回退出码 1，触发 `set -e` 导致脚本退出
- 输入读取逻辑使用了复杂的 `exec` 重定向

**解决方案**：
```bash
# 原代码（有问题）
((current_step++))

# 修复后
current_step=$((current_step + 1))
```

```bash
# 原代码（复杂）
exec 3<&0
exec 0</dev/tty
read -r REPLY
exec 0<&3

# 修复后
read -r REPLY < /dev/tty 2>/dev/null || read -r REPLY
```

---

### 2. 文件系统大小写敏感性问题

#### 问题：Git 克隆时文件冲突
**现象**：
```
warning: the following paths have collided...
'include/uapi/linux/netfilter/xt_CONNMARK.h'
'include/uapi/linux/netfilter/xt_connmark.h'
```

**原因**：Windows NTFS 文件系统默认不区分大小写，导致同名不同大小写的文件只能保留一个

**影响**：编译时出现头文件找不到的错误，如：
```
net/ipv4/netfilter/ipt_ECN.c:26:46: warning: 'struct ipt_ECN_info' declared inside parameter list...
```

**解决方案**：创建 ext4 虚拟磁盘

```bash
# 创建 20GB 虚拟磁盘文件
dd if=/dev/zero of=~/wsl-kernel-build/kernel-build.ext4 bs=1M count=20480

# 格式化为 ext4
mkfs.ext4 ~/wsl-kernel-build/kernel-build.ext4

# 挂载
mount -o loop ~/wsl-kernel-build/kernel-build.ext4 ~/wsl-kernel-build/mnt

# 在 ext4 分区中克隆内核源码
git clone --depth 1 --branch linux-msft-wsl-6.6.y \
  https://github.com/microsoft/WSL2-Linux-Kernel.git \
  ~/wsl-kernel-build/mnt/WSL2-Linux-Kernel
```

### 3. 磁盘空间不足

#### 问题：编译过程中空间耗尽
**现象**：
```
Filesystem      Size  Used Avail Use% Mounted on
/dev/loop0      9.8G  9.7G     0 100%
```

**原因**：初始只分配了 10GB 空间，但完整编译需要 15-20GB

**解决方案**：扩容虚拟磁盘

```bash
# 1. 卸载
umount ~/wsl-kernel-build/mnt

# 2. 扩容文件
truncate -s 20G ~/wsl-kernel-build/kernel-build.ext4

# 3. 扩容文件系统
e2fsck -f ~/wsl-kernel-build/kernel-build.ext4
resize2fs ~/wsl-kernel-build/kernel-build.ext4

# 4. 重新挂载
mount -o loop ~/wsl-kernel-build/kernel-build.ext4 ~/wsl-kernel-build/mnt
```

---

### 4. 内核模块未安装

#### 问题：Waydroid 安装时找不到 binder 设备
**现象**：
```
[!] 缺少设备: /dev/binder
[!] 缺少设备: /dev/hwbinder
[!] 缺少设备: /dev/vndbinder
modprobe: FATAL: Module binder_linux not found...
```

**原因**：
- 内核编译时启用了 binder 支持（built-in），但没有安装模块到 `/lib/modules/`
- 新内核的模块目录 `/lib/modules/6.6.123.2-microsoft-standard-WSL2+/` 不存在

**解决方案**：

```bash
# 编译模块
cd ~/wsl-kernel-build/mnt/WSL2-Linux-Kernel
make -j$(nproc) modules

# 安装模块
make modules_install

# 创建 binder 设备
mkdir -p /dev/binderfs
mount -t binder binder /dev/binderfs

# 创建符号链接
ln -sf /dev/binderfs/binder /dev/binder
ln -sf /dev/binderfs/hwbinder /dev/hwbinder
ln -sf /dev/binderfs/vndbinder /dev/vndbinder
```

---

### 5. ASHMEM 已过时

#### 问题：内核配置中 ASHMEM 未启用
**现象**：配置中找不到 `CONFIG_ASHMEM`

**原因**：从 Linux 5.18 开始，`ashmem` 被 `memfd` 替代，在 6.6 内核中已完全移除

**解决方案**：使用 `memfd` 替代
```bash
# 检查 memfd 是否启用
zcat /proc/config.gz | grep CONFIG_MEMFD_CREATE
# 输出: CONFIG_MEMFD_CREATE=y
```

---

### 6. 安装脚本 bug

#### 问题：内核路径解析错误
**现象**：
```
cp: cannot stat '[0;34m[INFO][0m 查找内核镜像...': No such file or directory
```

**原因**：`find_kernel_image()` 函数同时输出了日志和路径，导致变量包含颜色代码

**解决方案**：将日志输出重定向到 stderr
```bash
find_kernel_image() {
    log_info "查找内核镜像..." >&2  # 重定向到 stderr
    # ...
    printf '%s\n' "$kernel_image"  # 只输出路径到 stdout
}
```

---

### 7. 内核路径问题

#### 问题：WSL 无法加载中文路径的内核
**现象**：
```
找不到 C:\Users\15291\.wslconfig 中指定的自定义内核
错误代码: Wsl/Service/CreateInstance/CreateVm/WSL_E_CUSTOM_KERNEL_NOT_FOUND
```

**原因**：`.wslconfig` 中配置的路径包含中文字符，WSL 解析失败

**解决方案**：将内核复制到英文路径
```bash
# 复制到英文路径
mkdir -p /mnt/c/Users/15291/WSL2-Kernel
cp /mnt/d/番茄小说/wsl2-waydroid-kernel/build/bzImage-waydroid \
   /mnt/c/Users/15291/WSL2-Kernel/
```

`.wslconfig` 配置：
```ini
[wsl2]
kernel=C:\\Users\\15291\\WSL2-Kernel\\bzImage-waydroid
memory=8GB
processors=4
```

## 完整流程

### 1. 准备工作
```bash
# 创建 ext4 虚拟磁盘（20GB）
mkdir -p ~/wsl-kernel-build
dd if=/dev/zero of=~/wsl-kernel-build/kernel-build.ext4 bs=1M count=20480
mkfs.ext4 ~/wsl-kernel-build/kernel-build.ext4
mkdir -p ~/wsl-kernel-build/mnt
mount -o loop ~/wsl-kernel-build/kernel-build.ext4 ~/wsl-kernel-build/mnt
```

### 2. 获取内核源码
```bash
cd ~/wsl-kernel-build/mnt
git clone --depth 1 --branch linux-msft-wsl-6.6.y \
  https://github.com/microsoft/WSL2-Linux-Kernel.git
```

### 3. 配置内核
```bash
cd WSL2-Linux-Kernel
zcat /proc/config.gz > .config

# 启用 Waydroid 支持
echo "CONFIG_ANDROID=y" >> .config
echo "CONFIG_ANDROID_BINDER_IPC=y" >> .config
echo "CONFIG_ANDROID_BINDERFS=y" >> .config
echo "CONFIG_ANDROID_BINDER_DEVICES=\"binder,hwbinder,vndbinder\"" >> .config
echo "CONFIG_MEMCG=y" >> .config
echo "CONFIG_CGROUP_DEVICE=y" >> .config

make olddefconfig
```

### 4. 编译内核
```bash
make -j$(nproc)
make -j$(nproc) modules
make modules_install
```

### 5. 复制内核到 Windows
```bash
mkdir -p /mnt/c/Users/$USER/WSL2-Kernel
cp arch/x86/boot/bzImage /mnt/c/Users/$USER/WSL2-Kernel/bzImage-waydroid
```

### 6. 配置 WSL 使用新内核
创建 `C:\Users\<用户名>\.wslconfig`：
```ini
[wsl2]
kernel=C:\\Users\\<用户名>\\WSL2-Kernel\\bzImage-waydroid
memory=8GB
processors=4
```

### 7. 重启 WSL
```powershell
wsl --shutdown
```

### 8. 创建 binder 设备
```bash
mkdir -p /dev/binderfs
mount -t binder binder /dev/binderfs
ln -sf /dev/binderfs/binder /dev/binder
ln -sf /dev/binderfs/hwbinder /dev/hwbinder
ln -sf /dev/binderfs/vndbinder /dev/vndbinder
```

## 注意事项

1. **磁盘空间**：确保有足够的磁盘空间（建议 25GB+）
2. **大小写敏感**：Linux 内核源码必须在区分大小写的文件系统上编译
3. **路径问题**：避免在 `.wslconfig` 中使用中文字符
4. **模块安装**：编译内核后必须执行 `make modules_install`
5. **binder 设备**：每次 WSL 重启后需要重新挂载 binderfs
